## Đây là kiến thức Git bạn vừa học

Qua một lỗi nhỏ, bạn đã hiểu khá nhiều khái niệm quan trọng:

- `git remote -v` để xem repository đang kết nối bằng giao thức nào.
- Sự khác nhau giữa **HTTPS** (dùng thông tin đăng nhập/token) và **SSH** (dùng SSH key).
- `git remote set-url` chỉ thay đổi địa chỉ kết nối của remote `origin`, không ảnh hưởng đến repository trên GitHub.
- `ssh -T git@github.com` dùng để kiểm tra SSH key hiện tại đang xác thực với tài khoản GitHub nào.

Đây đều là những kiến thức rất hữu ích khi làm việc với Git và GitHub trong thực tế. Sau này khi làm nhiều dự án hoặc nhiều tài khoản, bạn sẽ biết cách xử lý thay vì phải clone lại repository từ đầu.
